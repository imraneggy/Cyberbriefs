Implementation Report

CyberBriefs Automation

A free-tier, serverless automation system for generating cybersecurity infographic posts with OpenAI, approving them in Telegram, and publishing approved posts to Instagram.

Executive Summary

The project is designed for a creator who does not want to run a VPS or dedicated server. Scheduled work runs on GitHub Actions, approval and publishing run on Cloudflare Workers, generated images are hosted on Cloudflare R2, and approval happens in Telegram.

Default Image Modelgpt-image-2
Output Formatjpeg for Instagram compatibility
Publishing GateNo Telegram approval, no Instagram post
Hosting CostDesigned for free-tier infrastructure

System Flow

GitHub Actions runs the morning or evening workflow.
Python selects a cybersecurity topic and asks OpenAI for copy, hashtags, and image prompt.
OpenAI Image API generates a JPEG infographic with gpt-image-2.
The image uploads to Cloudflare R2 and receives a public HTTPS URL.
Telegram sends the image and caption with approval buttons.
Cloudflare Worker receives the Telegram callback.
Approved posts publish through Instagram Graph API.

Tech Stack

  • Python 3.11 for generation scripts.
  • TypeScript Cloudflare Worker for webhook and publishing logic.
  • GitHub Actions for cron-based scheduling.
  • Cloudflare R2 for public image hosting.
  • Cloudflare KV for post state.
  • Telegram Bot API for manual approval.
  • Instagram Graph API for official publishing.
  • OpenAI Responses API and Image API for content creation.

Implementation Rundown

  1. Create the Cloudflare R2 bucket and KV namespace.
  2. Deploy the Cloudflare Worker and set Worker secrets.
  3. Register the Telegram webhook to the Worker URL.
  4. Add GitHub Actions secrets for OpenAI, Telegram, R2, and Worker access.
  5. Run the morning workflow manually and reject the first test post.
  6. Run the workflow again and approve the second post.
  7. Confirm Instagram publishing and Telegram confirmation.

Monetization Roadmap

  1. First 90 days: publish consistently and optimize for saves and shares.
  2. 1k-5k followers: add affiliate links for password managers, security courses, and labs.
  3. 5k-20k followers: sell awareness PDFs, templates, and sponsored posts.
  4. 20k+ followers: add newsletter sponsorships, B2B content packages, and consulting lead generation.

Known Constraints

  • OpenAI image generation is paid API usage, even when infrastructure is free-tier.
  • Instagram publishing requires a Business account linked to a Facebook Page.
  • Generated images must be publicly accessible by Meta crawlers.
  • GitHub scheduled workflows can run several minutes late on the free tier.
